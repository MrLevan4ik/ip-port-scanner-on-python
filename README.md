# IP Port Scanner on Python

Многопоточный сканер прокси с проверкой работоспособности, измерением скорости и丰富的 настройкой через CLI.

## Что умеет

- Проверяет **4 типа прокси**: SOCKS4, SOCKS5, HTTP, HTTPS
- Рекурсивно ищет все `.txt` файлы в указанной папке и её подпапках
- Измеряет **задержку** (мс) и **пропускную способность** (KB/s)
- Сохраняет результаты в **6 форматов**: TXT, CSV, JSON — для человека и для программы
- Многопоточная проверка с настраиваемым количеством потоков
- Логирует все ошибки в файл с возможностью быстрого открытия
- Корректно завершается по Ctrl+C с сохранением результатов
- Режим мониторинга с повторным сканированием по расписанию
- Автоматически скачивает публичные прокси-листы
- Определяет страну каждого прокси через бесплатный API

## Зависимости

```
pip install pysocks requests urllib3
```

| Пакет | Зачем |
|-------|-------|
| `pysocks` | Работа с SOCKS4/SOCKS5 прокси |
| `requests` | Проверка HTTP/HTTPS прокси + скачивание списков |
| `urllib3` | Подавление предупреждений о SSL |

## Быстрый старт

```bash
# 1. Установить зависимости
pip install pysocks requests urllib3

# 2. Положить файлы с прокси в папку proxies/
#    (формат: protocol://ip:port или ip:port)

# 3. Запустить
python main.py
```

Программа найдёт все `.txt` файлы в `proxies/`, проверит каждый прокси и выдаст результаты.

## Полная справка по CLI

Запустите `python main.py --help` для справки. Ниже — подробное описание каждой опции.

### Основные настройки

#### `-d`, `--input-dir` (по умолчанию: `proxies/`)

Папка, в которой лежат файлы со списками прокси. Программа рекурсивно обходит все подпапки и ищет файлы с расширением `.txt`.

```bash
python main.py -d /path/to/my/proxies/
```

#### `-o`, `--output-dir` (по умолчанию: текущая папка)

Куда сохранять 6 выходных файлов с результатами. Если папки нет — создаётся автоматически.

```bash
python main.py -o results/
```

#### `-t`, `--timeout` (по умолчанию: `5` секунд)

Максимальное время ожидания ответа от прокси. Уменьшайте для быстрой проверки большого списка, увеличивайте для нестабильных прокси.

```bash
python main.py -t 3    # быстро, но пропустит медленные
python main.py -t 15   # медленно, но поймает больше
```

#### `-w`, `--workers` (по умолчанию: `30`)

Количество параллельных потоков. Больше = быстрее, но больше нагрузка на сеть и CPU.

```bash
python main.py -w 100  # агрессивно, для мощных машин
python main.py -w 10   # мягко, для слабых машин
```

#### `--test-url` (по умолчанию: `http://httpbin.org/get`)

URL, который будет запрашиваться через прокси для проверки работоспособности и замера скорости. Можно заменить на любой доступный endpoint.

```bash
python main.py --test-url http://ifconfig.me
```

---

### Опциональные фичи

#### `--progress`

Включает прогресс-бар вместо вывода каждого прокси отдельной строкой. Показывает процент, количество проверенных/найденных, время и ETA.

```bash
python main.py --progress
# [====================----------] 63.2%  189/299  ok=47  12s elapsed  ETA 7s
```

#### `--retry N` (по умолчанию: `1`)

Количество повторных попыток для каждого прокси, если первая проверка не удалась. Полезно для нестабильных прокси, которые могут не ответить с первого раза.

```bash
python main.py --retry 3   # каждый мёртвый прокси проверяется 3 раза
```

#### `--min-speed MS`

Фильтр по максимальной задержке. Прокси с задержкой больше MS мс будут отброшены даже если работоспособны. Удобно, если нужны только быстрые прокси.

```bash
python main.py --min-speed 1000   # только прокси быстрее 1 секунды
```

#### `--limit N`

Ограничение на количество проверяемых прокси. Полезно когда в файлах тысячи записей, а нужно проверить только N штук (самые быстрые в итоге попадут в результат).

```bash
python main.py --limit 200   # проверить максимум 200 прокси
```

#### `--parallel-files N` (по умолчанию: `1`)

Сканировать N файлов параллельно вместо последовательного. Ускоряет работу когда много файлов.

```bash
python main.py --parallel-files 3   # 3 файла одновременно
```

#### `--validate-ip`

Проверяет формат IP-адреса (IPv4/IPv6) перед попыткой подключения. Отсеивает явно битые записи (например `abc:8080`) до того, как они попадут в потоки.

```bash
python main.py --validate-ip
```

#### `--geo`

Определяет страну каждого работающего прокси через бесплатный API `ip-api.com`. Результат добавляется во все выходные файлы. При большом количестве прокси может замедлить итоговое сохранение (лимит API: ~45 запросов/мин).

```bash
python main.py --geo
# В CSV появится колонка "country"
# В JSON появится поле "country"
```

#### `--fetch`

Перед сканированием скачивает публичные прокси-листы (SOCKS4, SOCKS5, HTTP) с GitHub и сохраняет в папку `-d`. Удобно для получения свежих списков без ручного поиска.

```bash
python main.py --fetch   # скачать списки и сразу проверить
```

#### `--fetch-urls URL [URL ...]`

Указывает дополнительные URLs для скачивания прокси-листов (вместе с `--fetch` или отдельно). Принимает список через пробел.

```bash
python main.py --fetch --fetch-urls https://example.com/proxies.txt https://other.com/list.txt
```

#### `--watch SEC`

Режим мониторинга. После завершения сканирования ждёт N секунд и запускает проверку заново. Работает до нажатия Ctrl+C. Удобно для отслеживания доступности прокси.

```bash
python main.py --watch 300 --progress   # проверка каждые 5 минут
```

---

### Логирование

#### `--no-log-errors`

Отключает запись ошибок в файл `logs/error.log`. По умолчанию все ошибки (таймауты, обрывы, невалидные прокси) логируются.

```bash
python main.py --no-log-errors
```

Лог-файл автоматически открывается, если ошибок много (порог: 10). Нажмите **Y** чтобы открыть.

## Формат входных данных

Программа понимает несколько форматов записей в `.txt` файлах:

```
# С протоколом:
http://192.168.1.1:8080
socks5://10.0.0.1:1080
socks4://172.16.0.1:4145
https://8.8.8.8:443

# Без протокола (тестируются все 4 типа):
192.168.1.1:8080
10.0.0.1:1080

# С указанием протокола через двоеточие:
192.168.1.1:8080:socks5

# Комментарии (пропускаются):
# Это комментарий
```

## Выходные файлы

Все файлы сохраняются в папку, указанную через `-o` (по умолчанию текущая).

### `1_all_working_proxies.txt`

Полный список всех работающих прокси, отсортированный от быстрых к медленным. Содержит адрес, задержку, скорость и исходный файл.

```
  1. socks5://1.2.3.4:1080                | 120.5ms (45.2 KB/s)    | proxies/data.txt
  2. http://5.6.7.8:8080                  | 230.1ms                | proxies/data.txt
```

### `2_proxies_for_reuse.txt`

Готовый список для копирования и использования в других программах. Только адреса в формате `protocol://ip:port`.

```
socks5://1.2.3.4:1080
http://5.6.7.8:8080
```

### `3_ips_only.txt`

Уникальные адреса `ip:port` без протокола. Удобно для импорта в инструменты, которые сами определяют тип прокси.

```
1.2.3.4:1080
5.6.7.8:8080
```

### `4_*_proxies.txt`

Отдельные файлы по каждому протоколу: `4_socks4_proxies.txt`, `4_socks5_proxies.txt`, `4_http_proxies.txt`, `4_https_proxies.txt`.

### `5_proxies_analysis.csv`

Таблица для открытия в Excel / Google Sheets. Содержит все поля: протокол, IP, порт, задержку, скорость, страну (если `--geo`), исходный файл.

### `6_proxies.json`

Структурированный JSON для программного использования. Каждый прокси — объект с полями `protocol`, `ip`, `port`, `response_ms`, `speed_kbps`, `country`, `source`.

```json
[
  {
    "protocol": "socks5",
    "ip": "1.2.3.4",
    "port": 1080,
    "response_ms": 120.5,
    "speed_kbps": 45.2,
    "country": "Germany",
    "source": "proxies/data.txt"
  }
]
```

## Логирование ошибок

Все ошибки (таймауты, обрывы соединения, невалидные прокси) записываются в `logs/error.log` с датой, временем и типом ошибки.

```
2025-07-12 14:30:01 | ERROR   | socks5://1.2.3.4:1080 — TimeoutError: ...
2025-07-12 14:30:02 | ERROR   | http://5.6.7.8:8080 — ConnectionError: ...
```

Если за одно сканирование произошло **10 или более ошибок**, программа предложит открыть лог-файл нажатием **Y**.

---

## Корректное завершение (Ctrl+C)

При нажатии Ctrl+C:
1. Программа перестаёт запускать новые проверки
2. Дожидается завершения уже запущенных
3. Сохраняет все найденные до этого прокси
4. Показывает итоговую статистику

При повторном нажатии Ctrl+C — принудительное завершение.

## Примеры использования

```bash
# Базовый запуск — проверить всё из папки proxies/
python main.py

# Своя папка, быстрая проверка
python main.py -d my_proxies/ -t 3 -w 50

# Прогресс-бар + повторные попытки
python main.py --progress --retry 3

# Только быстрые прокси, максимум 200
python main.py --min-speed 1000 --limit 200

# Параллельное сканирование + геолокация
python main.py --parallel-files 3 --geo

# Скачать свежие прокси-листы и проверить
python main.py --fetch

# Мониторинг каждые 5 минут с прогресс-баром
python main.py --watch 300 --progress

# Всё вместе: свои настройки, фильтры, гео, результаты в папку
python main.py -d proxies/ -o results/ -t 10 -w 100 \
  --progress --retry 3 --min-speed 500 --limit 500 \
  --geo --validate-ip
```
